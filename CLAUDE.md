## Writing conventions

- Always add an empty line below a markdown header (i.e. #, ##, ###, or ####)
- In Markdown, when you write something like "term - explanation", use colons instead of dashes. For example: "**foo:** bar" instead of "**foo** - bar"
- Always use British English spelling except for identifiers in code (e.g. variable or function names, which should be in American English spelling)
- Always use `# CLAUDE.md — <ProjectName>` as the main header in CLAUDE.md files
- Do not soft-wrap prose in files — write paragraphs as single unbroken lines
